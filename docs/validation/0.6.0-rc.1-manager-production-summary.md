## Self-hosting and runtime continuity

The prior Qdex failure was reproduced as
`runtime_mismatch_after_active_runtime_mutated`. The active Manager launch had
recorded the digest of mutable `qwendex_cli.py`; a valid self-edit changed those
bytes, so a later hook rejected the same runtime as mismatched. Immutable,
validated runtime generations were therefore required.

Acceptance proved that active sessions stay pinned, new sessions select the
current generation atomically, unvalidated candidates are rejected, failed
activation preserves the known-good generation, and shell/stock-Codex recovery
does not require a working Qdex session. Recovery ended with zero active or
stale agents and zero cross-generation hook calls.

## Claim ceiling and remaining limits

The certified boundary is Qwendex orchestration on tested Linux with the
canonical patched Codex `0.144.0` build. It is not an operating-system security
boundary. Enforced Manager guarantees require the supported patch; stock Codex
remains the Off-mode recovery path and supports non-native Qwendex CLI
functions. Manager lanes follow the turn plan rather than being universally
required for trivial work, and reasoning effort remains independent from
delegation mode.

The historical hardening candidate required default-branch, remote-CI,
annotated-tag, and publication gates. The final prerelease binds those gates to
the annotated tag rather than treating this historical acceptance as authority
for a changed source tree.
